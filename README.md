\# Лабораторна робота №3: AWS Infrastructure as Code (Terraform)



\## Опис проєкту

\[cite\_start]Цей репозиторій містить декларативні конфігураційні файли Terraform для автоматизованого розгортання безпечної мережевої інфраструктури та вебсервера на платформі Amazon Web Services (AWS)\[cite: 102]. 



\[cite\_start]Проєкт виконується згідно з параметрами \*\*Варіанту 17\*\*  та включає створення наступних компонентів:

\* \[cite\_start]Virtual Private Cloud (VPC) з діапазоном адрес `10.17.0.0/16`\[cite: 100, 111].

\* \[cite\_start]Дві підмережі (Subnets) у різних зонах доступності (`eu-central-1a` та `eu-central-1b`) для базової відмовостійкості\[cite: 100, 111].

\* \[cite\_start]Internet Gateway (IGW) та відповідні таблиці маршрутизації (Route Tables) для доступу до мережі Інтернет\[cite: 101].

\* \[cite\_start]Security Group (фаєрвол рівня інстансу), що дозволяє SSH-доступ виключно з поточної ІР-адреси адміністратора та відкриває доступ до вебсервера по кастомному порту `8081`\[cite: 101, 111, 284].

\* \[cite\_start]Amazon EC2 інстанс (на базі найновішого образу Ubuntu 24.04 LTS), який автоматично конфігурується під час завантаження за допомогою `user\_data` для встановлення та налаштування вебсервера Apache\[cite: 91, 285].



\[cite\_start]Файл стану інфраструктури (`terraform.tfstate`) надійно та ізольовано зберігається у віддаленому сховищі Amazon S3 з використанням нативного механізму блокування стану (State Locking) для запобігання конфліктам\[cite: 64, 72, 75].



\## Передумови (Prerequisites)

\[cite\_start]Для успішної роботи з даним репозиторієм локальне середовище має відповідати наступним вимогам:

1\. \[cite\_start]Встановлена утиліта \*\*AWS CLI v2\*\*\[cite: 116].

2\. \[cite\_start]Налаштований локальний профіль доступу до хмари AWS із дійсними Access Key ID та Secret Access Key (`aws configure`)\[cite: 131, 134].

3\. \[cite\_start]Встановлений інструмент \*\*Terraform\*\* (версія `1.10.0` або новіша)\[cite: 201].

4\. \[cite\_start]Попередньо підготовлений (через AWS CLI) S3 бакет із налаштованим версіонуванням та блокуванням публічного доступу для збереження `terraform.tfstate`\[cite: 145, 156, 166].



\## Інструкція з використання



\### 1. Ініціалізація (Init)

Перший запуск у новій директорії. Команда ініціалізує робочий простір Terraform, завантажує плагіни провайдера AWS та налаштовує підключення до віддаленого стану S3.

`bash

terraform init

`



\### 2. Валідація та планування (Validate \& Plan)

Перевірка синтаксису конфігурації та генерація плану виконання. Terraform зіставляє локальний код з наявним станом та формує список ресурсів на створення.

`bash

terraform fmt

terraform validate

terraform plan

`



\### 3. Розгортання (Apply)

Запуск процесу створення інфраструктури. Terraform автоматично розгорне всі описані ресурси в хмарі.

`bash

terraform apply

`

\*(Під час виконання команди система зупиниться і попросить підтвердити операцію вводом слова `yes`).\*



\### 4. Декомісіювання (Destroy)

Видалення всіх створених ресурсів. Це обов'язковий фінальний крок для мінімізації ризиків непередбачуваних витрат в AWS.

`bash

terraform destroy

`

\*(Після підтвердження Terraform прочитає файл стану з S3 і видалить усі компоненти у правильній зворотній послідовності).\*
